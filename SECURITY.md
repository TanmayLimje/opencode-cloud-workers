# Security Policy

## Supported Versions

The following versions of opencode-cloud-workers are currently being supported with security updates.

| Version | Supported |
| ------- | --------- |
| 0.1.x   | Yes       |

## Reporting a Vulnerability

If you discover a potential security vulnerability in this project, please report it via [GitHub Security Advisories](https://github.com/ManishModak/opencode-cloud-workers/security/advisories/new).

**Please do not report security vulnerabilities through public GitHub issues.**

### What qualifies as a security issue?

We take all security reports seriously. Examples of issues that qualify for a security report include:

- API key or credential exposure
- Authentication or authorization bypass
- Injection vulnerabilities (SQL, Command, etc.)
- Data leaks or privacy breaches

### Response Timeline

- **Acknowledgment**: You can expect an acknowledgment of your report within 48 hours.
- **Fix Target**: We aim to provide a fix or mitigation for confirmed vulnerabilities within 7 days of acknowledgment.

### Disclosure Policy

We follow a coordinated disclosure policy. We ask that you do not disclose the vulnerability publicly until a fix has been released and we have agreed on a disclosure timeline. Once the issue is resolved, we will publish a security advisory to inform the community.
